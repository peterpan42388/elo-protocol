# Legality

## 法律与合规边界（工程视角）

> 重要说明：本文件是工程边界说明，不构成法律建议。具体法务结论需由执业律师确认。

## 基本立场

1. ELO 是开源参考实现，不是自动等于可直接运营的商业金融服务。
2. 代码开源不免除使用者与运营者的合规责任。
3. 任何真实资金、托管、撮合、结算运营行为，都应先完成当地法律审查。

## 角色责任划分

1. 项目维护者（开源层）
- 负责代码透明、风险提示、文档边界说明。
- 不对第三方基于本项目进行经营的合规结果承担运营责任。

2. 二次开发者与运营者（落地层）
- 负责许可申请、AML/KYC、制裁筛查、数据合规等全部责任。
- 负责其商业产品的安全审计与用户保护机制。

3. 终端用户（使用层）
- 负责自行判断法域风险并遵守当地法律。

## 工程硬性要求

- 必须在仓库内持续维护并可见：
  - 项目免责声明
  - 安全披露渠道
  - 版本变更记录
- 涉及真实环境部署时，必须提供：
  - 配置隔离（密钥不入库）
  - 风险开关与回滚路径
  - 访问鉴权与审计日志

## 禁止误导表达

- 禁止将开源演示表达为“天然合规”或“自动合法”。
- 禁止对收益、安全、监管结果作保证性承诺。
- 禁止对用户提供未经授权的法律/投资建议。
