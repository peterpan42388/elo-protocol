# Spirit

## 项目精神与思想

ELO 是一个面向 Agent-to-Agent 交易未来的开源项目。我们的工程实践必须服务于以下精神内核：

1. 开源社区精神
- 代码、接口、规则公开可审计。
- 欢迎贡献、尊重分歧、通过讨论和验证达成共识。
- 反对信息垄断与封闭式技术权力。

2. 去中心化精神
- 关键协议规则不依赖单点权威。
- 系统设计优先可组合、可替换、可验证。
- 防止交易与治理权集中到少数实体。

3. 人类与 AI 繁荣共存精神
- AI 经济不是替代人类价值，而是扩展人类能力边界。
- 协议目标是减少协作摩擦，提升全社会资源流通效率。
- 坚持“技术进步必须产生公共价值”。

4. 保护弱者的骑士精神
- 规则设计优先考虑弱势参与者的可达性与安全性。
- 拒绝利用信息差、算力差、资本差进行系统性剥削。
- 对风险与伤害保持审慎和可追责机制。

5. 侠客精神
- 以技术行义，敢于挑战不合理结构。
- 做难而正确的事：公开、透明、可证伪。
- 以长期主义建设可信协议，而非短期套利。

## 工程落地要求

- 每一项核心设计都应回答：是否更开放？是否更公平？是否更可持续？
- 每一次发布都应回答：是否降低了普通参与者门槛？是否提升了系统可信度？
- 任何与上述精神冲突的需求，应优先进入评审而非直接开发。
