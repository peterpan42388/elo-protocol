# Review

Version: v0.1

所有提交 / PR 必须遵循以下审核执行原则。

## 1. 提交前检查

提交者必须在本地完成最小检查：
- 代码/文档变更与目标一致；
- 规则自检（Rule / Spirit / Target / Legality / Rejection）；
- 基础测试通过（按变更范围选择）。

## 2. 提交测试报告（强制）

每次提交必须附带一个与 Commit 号同名的 markdown 报告文件（例如 `16c37d6.md`），至少包含：
- 变更摘要
- 执行的测试与结果
- 规则遵守声明
- 风险点与回滚方案

若缺少该文件，默认拒绝提交。

## 3. 拒绝条件

- 若执行/提交记录违背 `Spirit.md` 或 `Target.md`：拒绝。
- 若执行/提交记录违背 `Legality.md`：拒绝，并同步更新 `Legality.md`。
- 若存在版本冲突且未完成本地合并验证：拒绝。

## 4. 审核机制

- 核心规则变更需至少一名维护者复核。
- 关键争议应在 PR 中留存可追溯讨论记录。
- 审核结论需可复现、可验证，而非主观拍板。
