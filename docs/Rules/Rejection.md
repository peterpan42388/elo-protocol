# Rejection

## 项目拒绝清单（不可接受事项）

以下事项与 ELO 项目原则冲突，默认拒绝：

1. 中心化后门
- 任何可绕过规则、任意冻结/挪用用户资产的隐藏管理能力。

2. 黑箱规则
- 无文档、无测试、无审计依据的核心交易逻辑。

3. 以牺牲弱势参与者为代价的效率优化
- 利用信息差或资源差进行系统性剥削。

4. 诱导性金融叙事
- 在无充分披露下宣传“保本”“稳收益”“无风险”。

5. 秘钥与敏感信息入库
- 私钥、助记词、密钥文件、明文凭据提交到仓库。

6. 破坏开源治理
- 绕过评审直接改动核心规则。
- 未经测试与文档更新发布破坏性变更。

7. 违法用途导向开发
- 明确用于规避监管、洗钱、制裁规避、欺诈等场景的功能请求。

## 处理机制

- 发现冲突需求时：进入治理评审，不直接实现。
- 发现违规提交时：立即阻断合并并触发安全审查。
- 重复违规参与者：依据治理规则限制权限。
