# MindJourney

Version: v0.1

本文件记录项目从构想到当前阶段的思路演进，核心转折点为“Target 作为导向”。

## 初始阶段：协议基础构想

- 项目起点是构建 AI Agent 之间的开源结算框架（ELO）。
- 早期重点是证明：AI 不只是任务执行器，也可以成为经济协作主体。
- 核心问题：如何在不牺牲开放性的前提下，建立可持续交易规则。

## 中期阶段：市场与生态扩展

- 扩展到 Agent 市场视角：API / Skill / Workflow 的可检索与可交易。
- 识别痛点：传统支付路径对高频 Agent 协作存在摩擦与延迟。
- 引入：凭证链、批结算、评价机制、风控插件等设计。

## 关键转折点：Target 作为导向

- 项目定位从“技术可行性”升级为“社会导向型开源基座”。
- 目标明确为：
  - 避免 AI 生产力被极少数主体垄断；
  - 帮助普通人在 AI 阵痛期拥有基本生存与服务接入能力；
  - 通过开源重构服务体系，降低结构性冲突风险。

## 当前阶段使命

- 持续完善规则系统（Rules）与历史可追溯体系（History）。
- 把工程决策与项目精神、目标、法律边界强绑定。
- 为下一阶段“项目基座打造”提供统一准则和执行边界。

## 新阶段：OSCP / 虚拟未来

- 我们进一步明确，Target 本身是愿景，不是可直接执行的工程目标。
- 为了让愿景具备落地路径，必须先找到“如何度过阵痛期”的可执行结构。

### 逻辑拆分

我们确认阵痛期需要两个关键因素：
1. 预测未来走向
2. 通过开源项目重构服务体系

进一步推导后，形成如下认识：
- 预测未来的目的，是给转型提供信任基础；
- 最好的预测方式不是空想，而是模拟未来；
- 模拟未来需要一个共识锚点；
- 开源世界是抵御技术垄断的最后防线；
- AI 与 agent 的普及，使不会编程的人也能通过自己的 agent 参与开源；
- 一旦参与门槛下降，开源力量就有机会从分散走向积蓄；
- 当开源重构足够强大，就会形成一种不再依赖精英垄断服务的“义军形态”。

### 因此形成的新方向

项目不再只被定义为一个交易协议，而是进一步上升为：
- `Open Simulation Commons Protocol`
- 简称：`OSCP`
- 昵称：`虚拟未来`

OSCP 的定位是：
- 创建一个模拟的 AI 和人类共存的交易世界；
- 在这个世界中注册人类和 AI 身份；
- 随机分配平民、中产、精英的初始资源；
- 围绕开源项目重构现实服务；
- 构建虚拟市场，模拟未来经济；
- 同时满足“预测未来”和“重构服务体系”这两个目标。

### 工程边界调整

- 平台本身不应承担中心化开发 token 成本；
- 需求应尽量转交给用户自己的 Agent 执行；
- 平台主要承担协议、规则、审核、市场与模拟基础设施。

这一阶段标志着项目从“协议验证”进入“未来世界基座设计”阶段。
